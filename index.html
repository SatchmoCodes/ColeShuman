ColorFill is a full stack game built with Remix. This project expanded on my static version of the game built with JavaScript by implementing 3 unique gamemodes, a global leaderboard, user authentication, and real time multiplayer.

Free Play mode lets you choose the size of the board and gives you a randomized board. Progressive mode gives you a small board that progressively gets bigger with the goal of getting as low of a score as possible. Player vs Player allows you to create a game and face another player in real time on the same board to see who can capture half the board first.

Scores are tracked and kept on the leaderboard as you play. From the leaderboard, you can play someone else's board they previously played and attempt to beat their score. In addition, personal stats are also tracked at the top, as well as global win rates, games played, and current win streaks.

Building this project broadened my experience in React and also gave me a great look at Remix. Working through a project of this scale made me overcome numerous obstacles, and forced me to dive deeper into many previously unheard of concepts, like Server Sent Events for example. As a reult of this, I feel much more confident working with both the Front and Back end of projects after completing this one.

Color Fill Game

CSS

JavaScript

OOP

Responsive

For this project, I used vanilla JavaScript to make a color fill game. Starting in the top left corner, the goal is to fill the entire square with the same color in as few turns as possible.

Building this project helped me refine my skills in Object Oriented Programming. I used a class to create and give properties to each individual square, and put those squares into a 2-dimensional array to make tracking them easier. I also got more experience working with ternary operators which helped make the code more concise.

To play the game, you click any of the 5 colored circles in the bottom to change the top left square, as well as any other squares you have captured. In the 3 circle row with the white buttons, the first resets the board, the second saves the current board when refreshing, and the third shows what squares you will capture in the next move after hovering over the colored circles.

A counter at the top keeps track of your turns. The export and import feature allows you to save the code for a board so you and a friend can play on the same board. There is a high score tracker as well as a counter for each type of square. Finally, there is a menu to change the color of the squares.

This project holds a list of humorous calendar entries and displays them in a carousel on the home page. The game page includes a multiple choice game that randomly grabs four titles and one piece of content and tracks your score. Finally, the view page shows an accurate calendar of the year as well as a table showing all the entries for the month.

Using a Python script, I grabbed the data for the entries off an Excel sheet and placed them into a JSON file. I used a fetch statement to asynchronously retrieve the data from the JSON file, and a whole host of other functions and event listeners to control the functionality of the pages.

The most fun and challenging part of this project was building the animations and making them look smooth. There are a lot of moving parts to an animation like that and getting every aspect of it to look right took a lot of perfecting.

For this project, I used vanilla JavaScript to recreate sudoku. I integrated a sudoku API from Dosuku to generate the numbers, solution, and difficulty level for each puzzle.

The project features an eraser, an option for note taking and a brush to highlight all of a specific number. Players can use either the built in numpad or their own keyboard to enter numbers. There is a built in timer and mistakes are tracked throughout each game.

Interacting with this API was a great learning experience. It helped me get more confident working with asynchronous data. Developing all the other features of this project was also a fun and challenging task.
